# 遛娃决定器 - 项目规格说明书

## 1. 项目概览

- **项目名称**: 遛娃决定器 (PlayDecider)
- **项目类型**: 移动端优先的 Web 应用 (PWA)
- **核心功能**: 本地化地点管理 + 每日随机决策生成器
- **目标用户**: 拥有 0-10 岁儿童的家长，面临决策疲劳，希望快速决定每日活动地点
- **核心价值**: "打开即用，一键决策"，通过极简交互减少认知负担

## 2. UI/UX 规格

### 2.1 布局结构

**首页（决策页）**
- Header: 极简标题 "遛娃决定器"，右上角设置图标
- Hero: 屏幕中央巨大交互区域，显示结果或"开始"按钮
- Footer: 显示当天剩余换选次数

**管理页（地点列表）**
- Header: "我的地点" 标题 + 返回按钮 + 添加按钮
- List Area: 卡片式列表，支持删除
- Empty State: 列表为空时显示温馨插画提示

### 2.2 视觉设计

**色彩方案**
- 主色 (Primary): `#FF8C66` (暖橙色) - 传达活力、快乐、温暖
- 背景色 (Background): `#FFFBF0` (暖米色) - 护眼，营造轻松氛围
- 文本色 (Text): `#4A4A4A` (深灰) - 柔和的阅读体验
- 强调色 (Accent): `#4ECDC4` (薄荷绿) - 用于成功提示或次要按钮
- 危险色: `#FF6B6B` (珊瑚红) - 用于删除操作

**字体**
- 标题: "Nunito", sans-serif (圆体字，显得亲切可爱)
- 正文: "Nunito", sans-serif

**间距系统**
- 基础单位: 4px
- 卡片间距: 16px
- 按钮高度: 最小 48px (适合手指点击)
- 页面内边距: 24px

**动效**
- 随机选择时文字滚动/跳动效果 (老虎机效果)，持续 1.5 秒
- 结果出现时有庆祝动画 (Confetti 彩带)
- 按钮点击时有缩放反馈
- 页面切换有滑动过渡

### 2.3 组件列表

1. **大行动按钮 (Big Action Button)**
   - 状态: 初始(开始)、随机中(滚动中)、结果展示
   - 样式: 巨大圆形按钮，直径 200px，带阴影

2. **地点卡片 (Place Card)**
   - 样式: 圆角卡片，白色背景，左侧地点图标，右侧删除按钮
   - 交互: 点击删除按钮显示确认

3. **添加输入框 (Add Input)**
   - 样式: 底部弹窗，圆角输入框 + 确认按钮
   - 验证: 输入不能为空，不能重复

4. **空状态 (Empty State)**
   - 样式: 温馨插画 + 提示文字 "先添加几个好玩的地方吧"

5. **换选按钮 (Reroll Button)**
   - 样式: 结果下方的小按钮，每天限制 3 次
   - 状态: 有次数时正常显示，次数用尽时禁用

## 3. 功能规格

### 3.1 数据存储
- 使用浏览器 `localStorage` 存储地点数据
- 无需登录，无需后端数据库
- 数据结构: `places`: string[]

### 3.2 地点管理

**添加地点**
- 用户输入地点名称（如"中央公园"、"楼下绘本馆"）
- 验证: 输入不能为空，不能重复
- 成功后自动关闭弹窗并更新列表

**查看地点**
- 列表展示所有已存地点
- 显示地点数量统计

**删除地点**
- 点击删除图标，弹出确认框
- 确认后移除地点

**预设数据**
- 首次进入时，系统提供 3 个默认选项：
  - "小区楼下"
  - "附近公园"
  - "超市"

### 3.3 每日决策

**触发**
- 用户点击主页"去哪玩？"按钮

**随机逻辑**
- 从 localStorage 列表中随机抽取 1 个项目
- 使用滚动动画展示随机过程

**展示**
- 动画结束后显示结果
- 显示庆祝动画

**防后悔机制**
- 结果下方提供"换一个"按钮
- 每天限制换选 3 次
- 次数用尽按钮变灰并显示"今天换选次数已用完"

### 3.4 边界情况处理

- **列表为空**: 点击随机按钮时，弹出提示"请先添加地点"
- **只有一个地点**: 直接显示该地点，跳过动画，提示"只有一个选项哦"
- **当天首次使用**: 换选次数初始化为 3

## 4. 验收标准

1. **即时性**: 网站加载时间在 1.5 秒以内
2. **流程闭环**: 用户从"打开网站"到"获得地点建议"不超过 2 次点击
3. **持久化**: 关闭浏览器再打开，之前输入的地点数据不丢失
4. **响应式**: 在 iPhone SE 到 iPhone 14 Pro Max 上布局均正常
5. **动画流畅**: 随机动画流畅，无卡顿

## 5. 技术栈

- **框架**: Vue 3 (Composition API)
- **构建工具**: Vite
- **样式**: Tailwind CSS
- **图标**: Heroicons
- **动画**: canvas-confetti (庆祝动画)
- **字体**: Google Fonts (Nunito)
